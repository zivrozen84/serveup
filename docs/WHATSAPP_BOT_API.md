# WhatsApp Bot API – הסבר וקישור למסעדה ספציפית

## 1. איך עובד WhatsApp Business API (Cloud API)

- **מה זה:** Meta (פייסבוק) מספקים **Cloud API** – שרתים שלהם שמקבלים ושולחים הודעות וואטסאפ מטעמך.
- **זרימה כללית:**
  1. **מסעדה** (או אתה) רושמים **מספר טלפון** ב-Meta Business (WhatsApp Business Account).
  2. המספר עובר **אימות** (SMS/שיחה) – אחרי זה הוא "מחובר" ל-API.
  3. **הודעות נכנסות** – Meta שולחים ל-**Webhook** (כתובת URL שלך) כל הודעה שמגיעה למספר הזה.
  4. **הודעות יוצאות** – האפליקציה שלך שולחת ל-API של Meta (עם Token) ו-Meta שולח את ההודעה לווואטסאפ של הלקוח.

כלומר: **מספר וואטסאפ אחד = ערוץ תקשורת אחד**. אתה מקשר את המספר הזה במערכת ל**מסעדה אחת** (למשל לפי `phone_number_id`), וכך כל הודעה שמגיעה למספר הזה מתורגמת לפקודות לעדכון **רק** המסעדה הזו.

---

## 2. קישור וואטסאפ אחד למסעדה ספציפית במערכת

### 2.1 מה צריך ב-Meta (פייסבוק)

- **WhatsApp Business Account (WABA)** – חשבון עסקי וואטסאפ.
- **אפליקציה** ב-[developers.facebook.com](https://developers.facebook.com) עם מוצר "WhatsApp".
- **מספר טלפון** מתווסף דרך:  
  **WhatsApp → API Setup → Add phone number**  
  אחרי אימות (SMS/קול) מקבלים:
  - **`phone_number_id`** – מזהה ייחודי של המספר ב-API (זה מה שנשמור במערכת).
  - **`display_phone_number`** – המספר בפורמט להצגה (למשל 972501234567).

חשוב: אם המספר כבר רשום ב-WhatsApp רגיל או ב-WhatsApp Business App, צריך **להסיר** אותו משם לפני שמוסיפים ל-Cloud API.

### 2.2 מה לשמור במערכת (במסד הנתונים)

כדי לקשר **וואטסאפ אחד ↔ מסעדה אחת**:

| שדה במערכת | תיאור |
|------------|--------|
| `whatsappPhoneNumberId` | ה-`phone_number_id` שהתקבל מ-Meta – **מזהה המספר** ב-API. |
| `whatsappAccessToken` (אופציונלי) | Token לגישה ל-API **של המסעדה הזו** – אם בעתיד תרצה שכל מסעדה תגדיר Token משלה. |

ב-**Prisma** אפשר להוסיף ל-`Restaurant`:

```prisma
model Restaurant {
  // ... שדות קיימים ...
  whatsappPhoneNumberId  String?   // מזהה המספר ב-WhatsApp Cloud API
  whatsappAccessToken   String?   // אופציונלי: Token ייעודי למסעדה
}
```

- **קישור:** מסעדה שמחוברת לבוט = יש לה `whatsappPhoneNumberId` (ו־אולי Token).
- **"בלעדי":** כל מסעדה עם מספר משלה – כל הודעה שמגיעה ל-`phone_number_id` שלה מטפלים רק בעדכונים של **המסעדה הזו** (צבעים, מנות, תמונות וכו').

## 3. Webhook – איך מגיעות ההודעות לאפליקציה

- Meta שולחים **כל** הודעה שנכנסת **לכל** המספרים שבאפליקציה ל-**אותו Webhook URL**.
- ב-**כל** בקשת POST ל-Webhook מקבלים ב-body את:
  - **`phone_number_id`** – מאיזה מספר נשלחה ההודעה.
  - **`from`** – מספר הלקוח (בעל המסעדה / משתמש).
  - **`text.body`** (או סוג הודעה אחר) – תוכן ההודעה.

**ניתוב למסעדה:**

1. קוראים מה-body את `phone_number_id`.
2. מחפשים במסד: `Restaurant` שבו `whatsappPhoneNumberId === phone_number_id`.
3. אם נמצא – הפקודה (למשל "שנה צבע ל-#ff0000") מתבצעת **רק על המסעדה הזו**.

כך וואטסאפ אחד ספציפי תמיד מקושר למסעדה אחת, ורק אליה משנים צבעים/מנות/תמונות.

### 3.1 אימות ה-Webhook (Verify)

- ב-Meta מגדירים **Callback URL** (ה-Webhook) ו-**Verify Token** (מחרוזת סודית שאתה בוחר).
- כש-Meta רוצים לאמת את ה-URL הם שולחים **GET** עם:
  - `hub.mode=subscribe`
  - `hub.verify_token=...` (ה-Token שהגדרת)
  - `hub.challenge=...`
- השרת שלך בודק ש-`hub.verify_token` תואם ל-Token השמור, ואם כן – מחזיר את `hub.challenge` ב-response. אחרי זה ה-Webhook מאומת.

## 4. שליחת הודעות (תשובות ללקוח)

- **Endpoint:**  
  `POST https://graph.facebook.com/v21.0/{phone_number_id}/messages`
- **Headers:**  
  `Authorization: Bearer {access_token}`
- **Body:**  
  JSON עם `messaging_product: "whatsapp"`, `to` (מספר נמען), `type` (למשל `text`), `text: { body: "..." }`.

ה-`access_token` יכול להיות:
- **אחד למערכת** – Token של האפליקציה ב-Meta, שמשמש את כל המספרים באפליקציה; או
- **לכל מסעדה** – אם שמרת `whatsappAccessToken` ל־Restaurant, משתמשים ב-Token של המסעדה לשליחה מהמספר שלה.

כך אתה יכול לענות ל-**בעל המסעדה** באותו וואטסאפ שבו הוא שלח פקודה (צבעים, מנות, תמונות).

---

## 5. המטרה: בעלי מסעדה משנים צבעים, מנות, תמונות (בלעדי)

- **"בלעדי"** = כל מסעדה מקושרת ל**מספר וואטסאפ אחד** (או יותר, אם תרצה בעתיד). כל פקודה שמגיעה למספר הזה משפיעה **רק** על המסעדה הזו.
- **זרימה:**
  1. בעל מסעדה שולח הודעה למספר הוואטסאפ של המסעדה שלו.
  2. Webhook מקבל את ההודעה עם `phone_number_id`.
  3. המערכת מחפשת `Restaurant` עם `whatsappPhoneNumberId === phone_number_id`.
  4. מפרשים את ההודעה כפקודה (למשל: "שנה צבע ראשי ל-#c2410c", "הוסף מנה סלט", "העלה תמונה למנה X").
  5. מעדכנים במסד רק את המסעדה הזו (צבעים, מנות, תמונות).
  6. שולחים תשובה ב-WhatsApp (למשל "הצבע עודכן" / "המנה נוספה").

כדי לתמוך בעריכת תמונות דרך הבוט – צריך לקבל מהמשתמש **תמונה** (סוג הודעה `image` ב-Webhook), לשמור אותה (למשל ב-storage / URL), ולעדכן את `imageUrl` של המנה במסד.

---

## 6. סיכום – קישור וואטסאפ אחד למסעדה

| שלב | פעולה |
|-----|--------|
| 1 | יצירת WABA ואפליקציה ב-Meta, הוספת מספר טלפון ואימות. |
| 2 | שמירת `phone_number_id` (ו־אופציונלי Token) במסעדה – שדות ב-`Restaurant`. |
| 3 | הגדרת Webhook ב-Meta ל-URL של השרת + Verify Token. |
| 4 | ב-Webhook: קוראים `phone_number_id` מה-body, מזהים מסעדה, מריצים פקודות רק עליה. |
| 5 | שליחת תשובות עם Token + `phone_number_id` ל-API של Meta. |

כך API של וואטסאפ בוט מקושר ל**וואטסאפ אחד ספציפי** (לפי `phone_number_id`) וכל לוגיקה של שינוי צבעים, מנות ותמונות מתבצעת **בלעדי** למסעדה שמספר הוואטסאפ שלה הוא זה.
